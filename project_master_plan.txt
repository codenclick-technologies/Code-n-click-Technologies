Code-n-Click Technologies - Enterprise Resource Planning (ERP) System
=====================================================================

1. PROJECT OVERVIEW
-------------------
This project is a comprehensive internal management system (ERP) designed for "Code-n-Click Technologies". It serves as a central hub for managing employees, HR processes, project management, and company resources. The system is built with a role-based architecture to ensure secure and appropriate access for different levels of staff.

2. USER ROLES & ACCESS LEVELS
-----------------------------
The system supports four distinct user roles:

A. OWNER (Super Admin)
   - Full access to the entire system.
   - "Executive Command Center" Dashboard.
   - View all financial data (Payroll, Revenue).
   - Manage all users (Employees, HR, Managers).
   - Advanced Analytics (Retention, Cost vs. Performance).
   - System-wide settings.

B. HR (Human Resources)
   - Employee Management (Onboarding, Offboarding, Profiles).
   - Attendance Monitoring (Daily logs, Corrections).
   - Leave Management (Review and Approve/Reject leave requests).
   - Recruitment (Post Jobs, Manage Applications, Interview Pipeline).
   - Payroll Processing (Generate slips, Manage salaries).
   - Company Policy Management.
   - Celebration Management (Birthdays, Anniversaries).

C. MANAGER
   - Team Management (View team members).
   - Task Management (Kanban board for assigning and tracking work).
   - Meeting Management (Schedule and view team meetings).
   - Performance Monitoring (Basic team stats).

D. EMPLOYEE
   - Personal Dashboard (Stats, Tasks, Attendance).
   - Attendance (Check-in/Check-out).
   - Leave (Apply for leave, View history).
   - Profile (View and edit personal details).
   - Resources (Access company documents and blogs).

3. KEY MODULES & FEATURES
-------------------------

3.1. AUTHENTICATION & SECURITY
   - Secure Login with JWT (JSON Web Tokens).
   - Role-Based Access Control (RBAC) protecting all routes.
   - Password Management (Change Password, Forgot Password flow).

3.2. DASHBOARDS
   - **Owner Dashboard**: High-level overview, financial widgets, recruitment pipeline, interactive "Deep Dive" modals for quick actions.
   - **HR Dashboard**: Operational focus on attendance, leaves, and employee directory.
   - **Manager Dashboard**: Focus on team productivity, tasks, and schedules.
   - **Employee Dashboard**: Personal productivity, attendance clock, and leave status.

3.3. EMPLOYEE MANAGEMENT
   - Centralized directory of all staff.
   - Detailed profiles including personal info, employment details, and emergency contacts.
   - Status management (Active, Inactive, Suspended).

3.4. ATTENDANCE SYSTEM
   - Real-time Check-in/Check-out.
   - Status tracking (Present, Absent, Late, Half-day).
   - Monthly attendance reports.

3.5. LEAVE MANAGEMENT
   - Leave Types: Sick, Casual, Earned, Unpaid.
   - Workflow: Employee applies -> HR/Owner approves or rejects.
   - Balance tracking.

3.6. RECRUITMENT (ATS - Applicant Tracking System)
   - **Jobs**: Create and publish job openings.
   - **Applications**: Receive applications via public career page.
   - **Pipeline**: Move candidates through stages (New -> Screening -> Interview -> Hired/Rejected).
   - Interactive modal for quick status updates.

3.7. PAYROLL & FINANCIALS
   - Salary structure definition.
   - Monthly payroll generation.
   - Payslip generation and download.
   - Interactive approval workflow for pending payrolls.

3.8. TASK MANAGEMENT
   - Kanban Board interface (To Do, In Progress, Review, Done).
   - Assign tasks to specific employees.
   - Priority levels (Low, Medium, High).
   - Real-time status updates.

3.9. COMMUNICATION & RESOURCES
   - **Messages**: Internal inbox for contact form submissions from the public website.
   - **Resources (CMS)**: Content Management System for internal blogs, tutorials, and documents.
   - **Meetings**: Schedule and manage team meetings.

3.10. SEO INFRASTRUCTURE (Organic Growth)
   - **Sitemap**: Auto-generated `sitemap.xml` for Google indexing.
   - **Robots.txt**: Crawler guidance file.
   - **Schema Markup**: JSON-LD structured data for "Organization" and "LocalBusiness".
   - **Meta Tags**: Dynamic, human-optimized titles and descriptions for every page.

4. TECHNICAL ARCHITECTURE
-------------------------

4.1. FRONTEND (Client)
   - **Framework**: React.js (Vite).
   - **Styling**: Tailwind CSS for responsive design.
   - **State Management**: React Context API (AuthContext).
   - **Routing**: React Router DOM.
   - **Charts**: Recharts for data visualization.
   - **Icons**: Lucide React.

4.2. BACKEND (Server)
   - **Framework**: NestJS (Node.js).
   - **Language**: TypeScript.
   - **Database ORM**: Prisma.
   - **Database**: PostgreSQL (Relational DB).
   - **Authentication**: Passport.js + JWT strategies.

5. WORKFLOW EXAMPLES
--------------------

A. HIRING A NEW EMPLOYEE
   1. HR posts a Job Opening via HR Dashboard.
   2. Candidate applies via Public Career Page.
   3. Application appears in "Recruitment" pipeline.
   4. HR/Owner interviews and moves candidate to "Hired".
   5. HR creates a new Employee account in "Employee Management".
   6. System generates Employee Code and temporary password.

B. PAYROLL PROCESSING
   1. Month end approaches.
   2. HR generates payroll for the month.
   3. Payroll entries appear as "Pending Approval".
   4. Owner reviews "Financials" on dashboard and clicks "Approve".
   5. Status updates to "Paid", and employees can download payslips.

C. DAILY WORKFLOW
   1. Employee logs in.
   2. Clicks "Check In" on Dashboard.
   3. Works on assigned Tasks (updates status on Kanban board).
   4. Clicks "Check Out" at end of day.
   5. System records total hours and status.
